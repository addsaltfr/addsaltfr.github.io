[版本更新] 快速回顧 微軟發布 KB5012592 (Windows 11 作業系統組建 22000.613)

發表於 2022-04-16 文章字數： 893 所需閱讀時間 ≈ 2 分鐘

前言

手上一部升級到 Windows 11 的機器，可以說幾乎是閒著的電腦，不久前的功能就是拿來寫寫 Windows 11 更新後觀察的文章所用，而今，受到疫情影響，這部機器可能要忙一陣子，所以，我認為還是得硬著頭皮更新。

也因此，Windows Update 走完更新後，來到 KB5012592 (作業系統組建 22000.613) 這裡。

更新前內容概覽

2022 年 4 月 12 日 - (作業系統組建 22000.613) KB5012592 是微軟官方的說明文件，只不過，坦白說，這一次我沒有細讀、更沒有等待太久，就把這支補丁打上去。

原因是…，最近幾支更新都算穩定，而讓我有點鬆懈。

然而，這次更新其實是有狀況的(詳可讀：Windows 11、Windows 10 更新先緩緩，最新版本爆 Chrome、Edge 無法使用災情 - 電腦王阿達)，只能說，我這次的運氣還不錯，這些災情沒有發生在我所使用的機器上。

一併打上其他的補丁

  1. April 12, 2022-KB5012121 Cumulative Update for .NET Framework 3.5 and 4.8 for Windows 11
  2. 使用 Windows 惡意軟體移除工具移除特定的常見惡意程式碼 (KB890830)
  3. 也包括上次我先不打上去的補丁：[版本更新] 快速回顧 微軟發布 KB5011563 (Windows 11 作業系統組建 22000.593)
  4. 附帶一提的是打完 KB5011563 這支補丁後，我特別去測試新功能：「當您搜尋設定小工具時，會返回對應的頁面。」也證實了我自己真的跟 Windows 11 實在太不熟，搞不清楚「返回對應的頁面」究竟幫了我什麼。

後記：期盼更好與共好能夠併存

做為個人電腦市佔率最高之作業系統，更新只要一有災情，影響的人是比較多的。

這次我算運氣好，但，我也不是常常運氣好。

我能夠理解，當一支程式或一套作業系統，發展到一定的程度時，要在每一次的更新出擊都達到完美，並不是那麼容易。

只是，做為一個 End User 而言，期待穩定、好使力的作業系統，應該不是太過份的期望。
