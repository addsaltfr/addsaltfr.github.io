[版本更新] 快速回顧 微軟發布 KB5013943 (Windows 11 作業系統組建 22000.675)

前言

5 月 10 日，微軟針對 Windows 11 釋出 KB5013943 (作業系統組建 22000.675) 的更新。

更新前內容概覽

截至本篇紀錄發稿前，官方的說明似乎還沒有繁體中文版，所以，有興趣的伙伴，可以小練一下英文：May 10, 2022—KB5013943 (OS Build 22000.675) 裡的說明。

而這一次，我會更新的原因是我遇到官方說明裡的狀況(如下)。

Components that rely on explorer.exe, such as File Explorer, the Start menu, and the taskbar, might be affected and appear unstable.

是這樣的，我狀況是我把一個在桌面上的檔案，拖拉放到檔案總管裡的資料夾裡，那個檔案就直接在畫面留下殘影，我本不介意，因為，這可能是類似我們平常讓手指碰到窗戶，留下來的指印之類的。

殊不知，我登出再登入，這個殘影還在，接著就是跑出一個 EXPLORER.EXE 無法運作之類的訊息，想開一個新的檔案總管也開不出來，工作列也跟著完全失靈，就剩開始功能表還沒有掛點，我索性就重開機來處理這個狀況，重開機之後，立刻開「設定」找到 Windows Update，跑一下更新，果不其然跳出 KB5013943 這支更新，我查找一下說明，發現我今天遭遇的狀況，應該就是需要這支補丁。

立馬更新

下載更新的速度有點慢，期間我因故需要重開機，一重開機就顯示電腦在更新，我就傻傻地等到…，用主機上實體按鈕直接把電腦重開機 XD。

大概來回 3 趟吧！才把這支補丁給打上去。

後記

這次更新前，我的檔案沒有因此掛點，算是萬幸，針對這支補丁，我認為電腦有空的時候，就趕快把它打上去，免得遇到比較大的災難，屆時無法挽回，那可就槽糕嚕！
