在 elementory OS 5.1 上輕鬆安裝嘸蝦米

嘸蝦米輸入法在 Linux 上面的安裝與使用，已經方便很多，基本無痛，對於喜歡挑戰的 Linux 玩家，可能，反而會覺得不是那麼好玩。

試了幾個 Linux Distro，我得到一個結論：就是只要選用的 Distro 容易裝上 fcitx 的話(目前只有 Fedroa 是例外)，要用嘸蝦米，最快且輕鬆的方式，就是直接用 fcitx。

Lubuntu 本是我在輕量化 Distro 的選擇，elementory OS 是在無意中發現的，安裝後，我發現我還滿喜歡的，原因是想要用什麼，基本上都要自己安裝，算是一個非常乾淨的系統，但有些人可能會說：好像 macOS，關於這一點，反正現在都像來像去，致敬來致敬去的，我個人沒有太多的價值判斷。

  1. 既然跟 ubuntu 系出同門，那麼，就不客氣地在終端機下 sudo apt-get install fcitx-table-boshiamy 這道指令，同樣地，一個指令，就讓相依的套件都安裝進來。
  2. 重新登入之前，我在終端機上面再下 im-config 這道指令。
  3. 其實，我在安裝之前，有先下一次 im-config 這道指令，有觀察到安裝完 fcitx 之後，系統就直接把輸入法平台預先設定到 fcitx，不過，這時還無法使用 fcitx，一定要重新登入才行。
  4. 不僅如此，重新登入後，嘸蝦米直接就是 Ctrl+空白鍵(Space) 後的預設中文輸入法。
  5. 嘗試一般的登打，沒有任何問題，另外，重新登入前，我其實也一併安裝 LibreOffice，這一套 Office 已經可以說是 Linux 上必裝的軟體。

fcitx 給了蝦米族非常大的便利，我猜想，那個嘸蝦米的 Table 檔應該不是官方推送給 fcitx 的。如果是這樣，究竟對行易有限公司的嘸蝦米有沒有侵權，我發現網路上有不少討論，其中有一篇是有人發函洽詢經濟部智慧財產局而得到的回應：智慧財產局著作權主題網-認識著作權-資料檢索-解釋資料檢索-電子郵件990322b，說明主要分為 2 點，我將第 2 點摘要如下。

經詢問專家學者結果，有認「嘸瞎米輸入法」的「字根表」、「字碼對照表」應屬著作權法所保護之語文著作或編輯著作者，因而建議 您在運用「嘸蝦米輸入法」的拆字規則撰寫其他輸入法程式時，應避免將前開字根表、字碼對照表重製於 您所撰寫的程式中，以免構成侵害他人著作之行為。

我的建議是與其擔心侵不侵權，那麼，直接買一套回家可能比較妥適。

再者，官方版本與 fcitx 版本，除了在輸入法圖示上的差別外，官方通常會給繁體中文、簡體中文與日文等總共 4 個表格檔，而 fcitx 這邊，似乎只有繁體中文。
