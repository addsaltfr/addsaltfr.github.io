解決重新安裝 macOS High Sierra 時，安裝程式卡關在錯誤訊息：「無法聯絡復原伺服器。」

發表於 2024-03-17 文章字數： 1.9k 所需閱讀時間 ≈ 3 分鐘

前言

目前手邊僅存的 Mac Mini（2014 年末），曾經，被拿來安裝過幾支不同的 Linux Distro。

但，我得說，好玩歸好玩，除非 macOS 都跑不動，不然，原汁原味的 macOS 還是比較合口味。

所以，僅管被我玩來又玩去，在大部分的時間裡，都是 macOS High Sierra 擔任主打系統。

砍掉重練

今天，我想說已經運轉好幾年，整個系統已經被我搞爛，只得砍掉重練，所以，重開機之後，就按住 Command + R，準備來一場重灌之旅。

想不到，天不從人願，給我在執行 macOS High Sierra 的安裝程式時，跳出「無法聯絡復原伺服器。」，在這個地方卡關。

解決方法

  1. 可開機安裝磁碟安裝法：一般遇到這樣的狀況，就是拿出可開機安裝磁碟(通常使用 USB 隨身碟製作)，重開機按住 C，進入可開機安裝磁碟就能夠迎刃而解。而這個方法可能會遇到的狀況，大概就是 USB 插槽或隨身碟壞掉。

  2. 網路復原安裝法：重開機，立即按住 Option + Command + R，更新到比較新的 macOS。

  3. 改變 NVRAM 軟體更新網址：這個解法，我個人沒有試過，主要是開啟 Terminal，先確認一下所使用的系統時間是否正確，再更改 NVRAM 軟體更新網址。

  4. 組合技解決法：先走「網路復原安裝法」，如果系統運行實在太慢，就在新系統裡製作可開機安裝磁碟，走「可開機安裝磁碟安裝法」，回到 macOS High Sierra。

後記

  1. 關於第 1 個解法，我所使用的是 SD 卡做為可開機安裝磁碟，然而我的 Mac Mini（2014 年末） 上面的 SD 讀卡機怪怪的，即使「偶爾」能夠讀得到 SD 卡，也無法順利從 SD 卡上開機。我也有想到 USB 的 SD 讀卡機，因為少用的關係，老早就送出去。又或者，先從其他 Mac 主機製作可開機安裝磁碟，就有機會搞定，但，Mac Mini（2014 年末）已是我現階段僅有的 Mac 主機，思來想去的結果，就是這個方法暫時是行不通的。

  2. 關於第 2 個解法：這一招我之前用過，而且，通常會更新到最新支援的系統，以 Mac Mini（2014 年末） 這部機器而言，最終支援的系統版本是 macOS Monterey，我之前曾把系統升級到較新版的 macOS Mojave 與 macOS Big Sur，無奈硬體性能幾乎都扛不住後續版本的系統，因此，映入眼簾的 macOS Monterey 著實讓我有點掙扎，也因此讓我想到後面會提到的「組合技解決法」。

  3. 關於第 3 個解法：我得知這個方法後，思考一下就決定這一次不走這個方法的原因是 http 並沒有加密，這樣的安裝，我認為有安全性相關的疑慮。至於，系統上寫入的 NVRAM 軟體更新網址本來是不是走 https，以我個人淺薄的經驗是無法去進一步考究的，只知道現在能夠走不加密的 http，是有機會成功復原到 macOS High Sierra。

  4. 關於第 4 個解法：這是我目前的走法，而且就現在而言，升級到 macOS Monterey 後，我感受到比較多的是整合上帶來的方便，其中一項是我在 iOS 上購買的幾支 App，現在也能夠在 macOS Monterey 使用，這感覺超爽的。再說，我也已經花費 990 元，購買嘸蝦米輸入法 X2，相信吃了還魂丹的 Mac Mini（2014 年末），拿來同時上網、聊天與看影片，只要負荷不太大，應該再戰個幾年，是沒有問題的。

  5. 原本系統預測要 2 小時 19 分鐘才能夠完成安裝，實際上不到 1 小時就能搞定，時間花費還是有點久(我的網路環境是「台灣大寬頻 1G」)。
